서울록(SeoulLog)

2. 활용 인공지능 학습용 데이터

활용 데이터명: 서울특별시의회 회의록
분야: 공공행정/정치
출처: 서울특별시의회 회의록 (https://ms.smc.seoul.kr/kr/assembly/main.do)

3. 핵심 내용

서울시 정보소통광장의 서울시의회 회의록 검색 기능을 개선해, 사용자가 원하는 내용을 더 빠르고 정확하게 찾을 수 있도록 돕는 서비스입니다.

주요 기능

(1) 안건 단위 검색
  • 회의록 전체가 아닌 개별 안건 단위로 검색 결과 제공
  • 관련 없는 내용이 포함된 회의록 전체를 확인할 필요 없음

(2) 시맨틱 검색 (의미 기반 검색)
  • 키워드 단순 매칭이 아닌 문맥과 의미를 이해하는 벡터 기반 검색
  • 의미상 관련도가 높은 결과만 정확하게 제공

(3) RAG 기반 질의응답 챗봇
  • 자연어로 질문하면 회의록 내용 기반으로 답변 제공
  • 예: "청년 취업 지원금 관련 안건 있어?" → 관련 안건 + 요약 제공

(4) 3줄 요약 및 쉬운 설명
  • 긴 회의록을 핵심 내용 3줄로 요약
  • 법률·행정 용어를 일상 언어로 변환
  • 요약 → 상세 → 원문 단계별 정보 제공

(5) 시각적 타임라인
  • 안건의 진행 과정 시각화 (제안 → 심의 → 의결 → 공포)


4. 제안 배경 및 목적

[제안 배경]

현행 시스템의 문제점

1. 낮은 검색 정확도
  • 키워드가 포함된 모든 회의록이 나열되어 실제 원하는 내용과 무관한 결과 다수 포함
  • 예: "교통" 검색 시 → 교통비, 교통정리, 교통사고 등 맥락 무관 결과 혼재

2. 회의록 단위 검색의 한계
  • 검색 결과가 회의록 전체 단위로 제공되어 원하는 내용의 위치 파악 불가
  • 전체 문서를 다운로드하여 재검색 필요 (평균 5~10분 소요)

3. 비효율적인 정보 접근
  • 법률·행정 용어로 인한 이해 어려움
  • 안건의 현재 진행 상태 파악 불가
  • PDF/HWP 형식으로만 존재하여 데이터 재활용 불가능

[목적]

서울시 정보소통광장에서 필요한 정보를 정확하고 빠르게 찾을 수 있는 검색 환경을 구축합니다.

1. 검색 정확도 향상: 의미 기반 검색으로 관련도 높은 결과만 제공
2. 검색 효율성 증대: 안건 단위 검색으로 원하는 정보 즉시 접근
3. 정보 접근성 개선: 요약, 쉬운 설명, 챗봇으로 다양한 접근 방식 제공
4. 데이터 구조화: 회의록을 안건 단위로 파싱하여 재활용 가능한 형태로 변환


5. 세부 내용

[시스템 구조]

서울특별시의회 회의록 (data)
    ↓
회의록 원문 데이터 (파싱)
    ↓
안건 단위 분리 + 벡터 DB 저장 (ChromaDB)
    ↓
검색 인터페이스 + RAG 챗봇 (Next.js)

주요 구현 방법

(1) 회의록 크롤링 및 파싱
  • 기술: Python + BeautifulSoup4
  • 회의록 구조 분석하여 개별 안건 추출 후 JSON 형식으로 저장
  • 현황: 9개 회의록, 636개 안건 파싱 완료

(2) 벡터 데이터베이스 구축
  • 기술: ChromaDB + OpenAI text-embedding-3-small/large
  • 각 안건 텍스트를 벡터로 임베딩하여 유사도 검색 가능하게 저장
  • 메타데이터 함께 저장: 회의명, 날짜, 카테고리, 지역 등

(3) 시맨틱 검색
  • 시맨틱 검색: ChromaDB 하이브리드 검색 (벡터 + 메타데이터 필터)
  • 검색어를 벡터로 변환 → 코사인 유사도 계산 → 관련도 높은 순 정렬
  • 예: "지하철 요금 인상" 검색 시 관련 정책 안건만 의미적으로 매칭

(4) RAG 챗봇
  • 기술: LangChain/LangGraph + LLM API (OpenAI GPT/Gemini)
  • 사용자 질문 → 벡터 검색으로 관련 안건 추출 → LLM이 컨텍스트 기반 답변 생성 → 출처 함께 제공

(5) 프론트엔드
  • 기술: Next.js 16 + TypeScript + Tailwind CSS
  • 주요 페이지: 메인(안건 목록), 상세(요약/타임라인), 검색, 챗봇



개발 단계
  • Phase 1 (완료): 데이터 수집 및 UI 구현
  • Phase 2 (예정): ChromaDB 설정 및 시맨틱 검색 구현
  • Phase 3 (예정): RAG 챗봇 구현
  • Phase 4 (예정): LLM 기반 자동 요약 및 최적화


6. 기대 효과

[정량적 효과]

(1) 검색 정확도 향상
  • 현재: 키워드 매칭 방식, 정확도 약 40~50%
  • 목표: 시맨틱 검색으로 정확도 85% 이상

(2) 검색 시간 단축
  • 현재: 검색 → 다운로드 → 문서 내 재검색 (평균 5~10분)
  • 목표: 검색 → 즉시 해당 안건 확인 (평균 30초)
  • 10~20배 시간 단축

(3) 사용자 만족도
  • 검색 성공률: 50% → 85% (1.7배 ↑)
  • 페이지 체류 시간: 30초 → 3분 (6배 ↑)
  • 재방문율: 5% → 30% (6배 ↑)

(4) 데이터 활용도
  • 구조화된 데이터로 변환하여 API 제공, 데이터 분석, 타 서비스 연계 가능

[정성적 효과]

(1) 정보 접근성 강화
  • 법률·행정 용어에 익숙하지 않은 일반 시민도 쉽게 이해
  • 챗봇을 통한 대화형 정보 탐색 가능

(2) 행정 효율성 증대
  • 공무원, 연구원 등의 안건 검색 시간 절감
  • 유사 안건 및 선례 검색 용이

(3) 민주주의 참여 활성화
  • 시의회 활동에 대한 시민 관심 증가
  • 정책 결정 과정의 투명성 향상

(4) 확장 가능성
  • 타 지역 의회(경기도, 부산 등) 및 국회로 확장 가능
  • Open API 제공으로 시민단체, 언론사 등의 2차 활용 지원
